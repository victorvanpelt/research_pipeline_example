# AGENTS.md — context for AI coding agents

This repository is a template for a reproducible research pipeline. Data flows through numbered folders; everything downstream of the raw data is regenerated by code.

## Rules

- Never modify the raw data in `0_data/`. It is read-only. The exception is `0_data/codebook.md`, which is hand-maintained documentation: update it whenever the data change.
- Never edit files in `2_process/` or `3_output/` by hand (including `3_output/data_appendix/`): they are build artifacts, regenerated by the pipeline.
- Help the user make changes in `1_code/` (analysis) or `4_drafts/` (paper, slides, and data appendix sources) only.
- Before a change that touches more than one file, say which files you will edit and what will change in each. Then edit.
- The three analysis engines must stay in sync: `1_code/code.do` (Stata), `1_code/code.r` (R), and `1_code/code.qmd` (Quarto) produce identical results. Exception: the data appendix output is written by `code.r` and `code.do` only; `code.qmd` is a self-contained alternative and skips it.
- Never run `git commit` or `git push`. Leave changes in the working tree for the maintainer to review and commit.
- Do not commit credentials; use environment variables.
- Remove any scratch or helper files you create. The folder structure is fixed; do not add folders or top-level files.

## Build

- Full rebuild: `make`
- Single steps: `make r`, `make stata`, `make quarto`, `make appendix`, `make paper`, `make slides`
- Reset intermediates and outputs: `make clean`

## Verification

A change is done only when `make clean` followed by `make` completes without errors and every number in the `4_drafts/` outputs comes from the rerun pipeline, not from hand edits.

Verify before you report:

1. After the rebuild, check that the PDFs in `4_drafts/` were regenerated.
2. `make` runs the R engine only. If you edited `1_code/code.do` or `1_code/code.qmd`, also run `make stata` or `make quarto`; if the tool is not installed, say so instead of guessing.
3. Run `git status` and confirm that only the files you meant to change are modified.
4. Report what changed, whether the rebuild succeeded, and anything you could not verify.

Do not report a change as done without having rebuilt. "It should work" is not verification.
